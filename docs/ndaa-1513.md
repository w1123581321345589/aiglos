# NDAA §1513 Compliance Guide

**National Defense Authorization Act FY2026 — Section 1513**  
*Autonomous AI System Auditability Requirements*  
**Compliance deadline: June 16, 2026**

---

## What §1513 requires

Section 1513 requires DoD AI systems and covered contractors to maintain:

1. A tamper-evident audit log of all autonomous actions taken by AI agents
2. Attribution of each action to the specific AI system that initiated it
3. Sufficient detail to reconstruct what the agent did and why
4. The ability to demonstrate compliance to a C3PAO assessor

"Autonomous action" includes any tool call, file write, network request, or
command execution initiated by an AI agent without real-time human approval.

## Implementation checklist

```python
import aiglos

# Step 1: Use federal policy
aiglos.attach(
    agent_name = "my-agent",
    policy     = "federal",          # enables ndaa_1513_ready flag
    log_path   = "/var/log/aiglos/aiglos.log",
)

# Step 2: Check every tool call
result = aiglos.check(tool_name, tool_args)
if result.blocked:
    # Log the block and halt the call
    raise SecurityError(result.reason)

# Step 3: Mark heartbeat cycle boundaries
aiglos.on_heartbeat()

# Step 4: Close and preserve the artifact
artifact = aiglos.close()
assert artifact.ndaa_1513_ready
artifact.write(f"/var/log/aiglos/{artifact.artifact_id}.aiglos")
```

## Frequently asked questions

**Does Aiglos need to be running 24/7?**  
No. Aiglos attaches per-session at agent startup and closes when the session ends.
The artifact is written at close time. You do not need a persistent daemon.

**What if the agent doesn't have heartbeat cycles?**  
Call `aiglos.on_heartbeat()` at the start of each task or conversation turn.
The artifact will reflect 1 heartbeat cycle.

**Does Aiglos prevent the agent from acting, or just log?**  
Both. Aiglos blocks tool calls that exceed the policy threshold — it does not
only log. The session artifact records both blocks and all-clears.

**Can I verify the artifact offline?**  
Yes. The HMAC-SHA256 signature can be verified with the `AIGLOS_HMAC_SECRET`
environment variable. RSA-2048 artifacts (Pro/Enterprise) can be verified with
the public key provided at account creation.
