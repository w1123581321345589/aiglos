The problem

Coding agents don't use MCP.
They use your shell.

Cursor, Claude Code, Copilot, and Cline execute commands directly. They run subprocess.run(), call REST APIs with requests.post(), and write files across your entire codebase. Existing security tools were built for MCP. They miss everything else.

Amazon's own AI coding tool ran terraform destroy followed by terraform apply on a production environment. Both commands individually valid. Together, a 13-hour outage. Amazon's fix: mandatory senior engineer sign-off on all AI-assisted code. A human workaround to a software problem.

Blast radius control

Not all commands
are created equal.

Aiglos classifies every subprocess call into one of three tiers before execution. The tier determines how the action is handled — automatically, without requiring security team involvement for routine operations.

Tier Action type Examples Behavior
Tier 1 Autonomous git status, git log, ls, cat, pip list, pytest Auto-allow · logged
Tier 2 Monitored git commit, pip install, npm install, file writes Allow · compensating transaction logged
Tier 3 Gated rm -rf, terraform destroy, kubectl delete, DROP TABLE, git push --force Block · or pause + webhook approval

Tier 3 behavior is configurable per deployment: block (hard exception), pause (emit PagerDuty/Slack webhook, resume on signed approval token), or warn (log and allow). Default for enterprise: pause.
