Aiglos vs ClawKeeper

Both are open source. Both integrate with the OpenClaw ecosystem. They solve completely different problems — and together they're the most complete security posture available.

⚡
The short answer
ClawKeeper scans your OpenClaw host configuration before deployment. Aiglos monitors what your agent actually does at runtime. ClawKeeper tells you the CVE exists. Aiglos tells you whether your agent triggered it. Both tools are necessary. Neither replaces the other.

The architectural difference

ClawKeeper operates at the configuration layer — it answers "is your OpenClaw deployment hardened correctly?" before anything runs. This is host security: firewall rules, file permissions, CVE versions, credential storage.

Aiglos operates at the runtime layer — it answers "is your agent doing what it's supposed to be doing right now?" This is behavioral security: tool call sequences, anomaly detection, policy enforcement, compliance artifacts.

The gap neither tool covers alone: A perfectly hardened OpenClaw deployment (ClawKeeper A grade) can still run an agent that's been compromised via prompt injection, RAG poisoning, or supply chain attack. Aiglos catches the behavioral deviation after initial access — which ClawKeeper's static checks cannot see.
The gap Aiglos doesn't cover without ClawKeeper: A perfectly behaving agent running on an unhardened host with exposed ports and default credentials is a breach waiting to happen. ClawKeeper catches the configuration exposure before the agent ever runs.

Using both together

Aiglos ingests ClawKeeper badge data from the Skills Hub and incorporates it into the SkillReputationGraph. A skill flagged as Critical by ClawKeeper automatically elevates Aiglos's detection sensitivity for agents using that skill.

Run a unified audit combining both:

# ClawKeeper host audit
clawkeeper audit --json > host-audit.json

# Aiglos runtime + clawkeeper merge
aiglos audit --deep --clawkeeper host-audit.json
✓ Unified Report
The merged output combines ClawKeeper's 44 host hardening checks with Aiglos's runtime behavioral findings and NDAA §1513 signed artifact in a single report. The only format that covers both the infrastructure layer and the behavioral layer simultaneously.

Which should you install first?

If you're not running OpenClaw yet: install both before your first deployment. ClawKeeper to harden the host, Aiglos to monitor the runtime.

If you're already running OpenClaw: Aiglos first. You get immediate visibility into what your running agents are doing. Then use ClawKeeper to harden the underlying infrastructure.

If you're deploying for NDAA §1513 compliance: both are required. ClawKeeper satisfies host hardening evidence. Aiglos produces the signed session artifact. Neither alone satisfies the full requirement.
