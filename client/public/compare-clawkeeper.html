Architecture

Aiglos

Runtime behavioral intelligence

Aiglos is an in-process SDK that intercepts tool calls, HTTP requests, and subprocess commands before they execute. It builds a behavioral baseline of your agent's normal activity and detects deviations in real time.

  • Patches MCP ClientSession at import time
  • Synchronous scanning — under 1ms per call
  • Continuous protection for the full agent session
  • Federated intelligence across deployments
ClawKeeper

Point-in-time host scanner

ClawKeeper scans the host environment for known misconfigurations and vulnerable files. It runs as a CLI tool, produces a report, and exits. It does not monitor runtime behavior.

  • Checks 11 static signals on the host
  • Produces a pass/fail report
  • Runs once — no continuous monitoring
  • No awareness of agent tool call behavior

When to use each

Use Aiglos when

  • Your AI agents make tool calls you need inspected at runtime
  • You need behavioral anomaly detection across sessions
  • You need compliance evidence (NDAA §1513, EU AI Act, NIST)
  • You want campaign-level threat detection
  • You need outbound secret scanning and honeypot detection
  • You want a security grade (GOVBENCH) for your deployment

Use ClawKeeper when

  • You need a quick scan of your host environment
  • You want a pass/fail check before deploying
  • You need to verify static file integrity
  • You want a lightweight pre-flight check

Better together

Aiglos and ClawKeeper are complementary. Aiglos ingests ClawKeeper badge data (13,000+ skills) to enrich its runtime decisions. Use ClawKeeper to scan your host before deployment, and Aiglos to protect every tool call once your agents are running.
