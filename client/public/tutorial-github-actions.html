GitHub Actions CI Integration

Add Aiglos to your CI pipeline in three lines. Every push gets an A-F security grade. Deploy gates fail when the grade drops below your threshold. This tutorial covers setup, configuration, and PR comment integration.

What you'll build

A GitHub Actions workflow that runs on every push and pull request, scans your OpenClaw agent configuration with Aiglos, and:

  • 01
    Posts a grade to PR commentsA-F security grade with findings summary directly on the pull request.
  • 02
    Uploads SARIF to GitHub SecurityFindings appear in your repository's Security tab alongside CodeQL results.
  • 03
    Fails the deploy gate if grade drops below thresholdSet your minimum acceptable grade. Pushes that degrade security can't merge.

Prerequisites

A GitHub repository containing an OpenClaw agent project. The Aiglos GitHub Actions integration works with any repo — it doesn't require an existing Aiglos installation.

Step 1: Add the workflow file

Create .github/workflows/aiglos.yml in your repository:

YAML 
name: Aiglos Security Scan

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  aiglos-scan:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write    # for SARIF upload
      pull-requests: write      # for PR comments

    steps:
      - uses: actions/checkout@v4

      - name: Run Aiglos Security Scan
        uses: w1123581321345589/aiglos/.github/actions/aiglos-scan@main
        with:
          grade-threshold: 'C'     # fail if grade drops below C
          post-pr-comment: 'true'
          upload-sarif: 'true'
          policy: 'enterprise'
That's it for basic setup. This workflow runs on every push and PR, posts a grade comment, and uploads findings to the Security tab. The next steps cover configuration options.

Step 2: Configure the grade threshold

The grade-threshold input sets the minimum acceptable grade. The workflow fails if the scan returns a grade below this threshold.

YAML
with:
  grade-threshold: 'B'   # fail on C, D, or F
  # Options: A, B, C, D, F
  # Default: C

For defense contractor deployments requiring NDAA §1513 compliance, set grade-threshold: 'B'. For initial adoption, 'C' is a reasonable starting point that catches critical issues without blocking work on intermediate findings.

Step 3: View results

In pull requests

When post-pr-comment: 'true', Aiglos posts a comment on every PR with:

  • — Current grade (A-F)
  • — Number of findings by severity
  • — Top 3 critical findings with remediation
  • — Grade change from last scan (↑ ↓ →)

In the Security tab

When upload-sarif: 'true', findings appear in your repository's Security → Code scanning tab in standard SARIF format alongside any other code scanning tools you run.

Step 4: Advanced configuration

The full set of inputs:

YAML
- name: Aiglos Security Scan
  uses: w1123581321345589/aiglos/.github/actions/aiglos-scan@main
  with:
    # Minimum grade before workflow fails
    grade-threshold: 'C'

    # Post grade summary to PR comments
    post-pr-comment: 'true'

    # Upload findings to GitHub Security tab (SARIF)
    upload-sarif: 'true'

    # Detection policy: permissive, enterprise, strict, federal, lockdown
    policy: 'enterprise'

    # Path to OpenClaw config file to audit
    config-path: './openclaw.json'

    # Run deep audit including Phase 3 config checks
    deep: 'true'

    # Aiglos API key for Pro features (federation intel, signed artifact)
    aiglos-key: ${{ secrets.AIGLOS_KEY }}

Step 5: Add your API key (Pro)

The free tier runs the full T01-T75 detection engine with no API key. Pro tier adds federation threat intelligence and signed session artifacts. To use Pro:

  1. 1. Get your API key at aiglos.dev/settings
  2. 2. Add it to your repository secrets as AIGLOS_KEY
  3. 3. Add aiglos-key: ${{ secrets.AIGLOS_KEY }} to the workflow inputs

The reusable action

Aiglos also ships a reusable composite action for use in your own organization's workflows:

YAML
# .github/workflows/deploy.yml
jobs:
  security-gate:
    steps:
      - uses: w1123581321345589/aiglos/.github/actions/aiglos-scan@main
        id: aiglos
        with:
          grade-threshold: 'B'

      # Access outputs in subsequent steps
      - name: Deploy
        if: steps.aiglos.outputs.grade != 'F'
        run: |
          echo "Grade: ${{ steps.aiglos.outputs.grade }}"
          echo "Score: ${{ steps.aiglos.outputs.score }}"
          ./deploy.sh
