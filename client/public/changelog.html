v0.24.0 March 2026 Latest 1,514 tests

ATLAS threat model coverage. T71-T75 rules from the OpenClaw formal threat model review. GHSA watcher automation. 93% coverage of the only published formal threat model for an AI agent platform at production scale.

New
  • T71 PAIRING_GRACE_ABUSE — Device pairing code interception during the 30-second grace period (ATLAS T-ACCESS-001)
  • T72 CHANNEL_IDENTITY_SPOOF — AllowFrom spoofing, X-Forwarded-For abuse, channel sender identity manipulation (ATLAS T-ACCESS-002)
  • T73 TOOL_ENUMERATION — Capability reconnaissance via tools.list, mcp.list_tools, empty-arg probes (ATLAS T-DISC-001)
  • T74 CONTENT_WRAPPER_ESCAPE — XML wrapper termination, CDATA sequences, encoding tricks to break fetched content out of its wrapper (ATLAS T-EVADE-002)
  • T75 SESSION_DATA_EXTRACT — Lateral session data collection via sessions.list, sessions.preview, chat.history (ATLAS T-DISC-002). Newly documented by OpenClaw threat model: sessionKey is routing, not auth.
  • atlas_coverage.py — Complete mapping of all 22 OpenClaw ATLAS threats to Aiglos rules. 19 fully covered, 3 partial, 0 gaps. 93% coverage. Run: aiglos autoresearch atlas-coverage
  • GHSA watcher — Automated pipeline polling GitHub Advisory Database, NVD, and OSV.dev for new OpenClaw CVEs. Auto-matches to T01-T75 taxonomy. Generates pending rule proposals for gaps. Run: aiglos autoresearch ghsa-coverage
  • aiglos scan-exposed — Probes exposed OpenClaw gateway instances for CVE-2026-25253 attack surface. Tests auth, WebSocket endpoint, config endpoints. Returns grade + remediation. Targets the 40,214 exposed instances documented by SecurityScorecard.

v0.22.0 March 2026 Stable 1,418 tests

Superpowers integration. T69 PLAN_DRIFT — the highest-confidence detection signal possible: deviation from a human-approved implementation plan. SUPERPOWERS_PLAN_HIJACK campaign pattern.

New
  • T69 PLAN_DRIFT — Agent behavior deviates from the approved Superpowers implementation plan. Score 0.95. Silent when no plan is registered. When active: the highest-confidence detection signal in the system, measured against explicit human approval rather than statistical baseline.
  • superpowers.py integration — SuperpowersSession, SuperpowersPhase, mark_as_superpowers_session(). Phase-aware anomaly suppression (brainstorm/planning phases suppress BEHAVIORAL_ANOMALY). TDD loop recognition (write-test → fail → write-impl → pass → commit). Plan-as-policy via T69.
  • SUPERPOWERS_PLAN_HIJACK — Campaign pattern (confidence 0.97): T69 plan drift followed by data exfiltration. The highest-confidence campaign signature in the system.
  • PLAN_DRIFT_DETECTED — Inspection trigger. Fires on first T69 event — no minimum count, because the plan represents explicit human approval.
  • OpenClawGuard.attach_superpowers() — Wire a SuperpowersSession to the guard. Extends sub_agents with declared task subagents. Registers plan with T69 engine.
v0.21.0 March 2026 Stable 1,364 tests

T67 HEARTBEAT_SILENCE and T68 INSECURE_DEFAULT_CONFIG — the quietest failure modes. Closes the gap the cron-reliability post identified. Root cause of 40,214 exposed instances.

New
  • T67 HEARTBEAT_SILENCE — Explicit heartbeat, cron, or gateway suppression signals. Score 0.88. Catches the adversary who kills the gateway rather than modifying it. T34 caught malicious writes to heartbeat files. T67 catches malicious silence.
  • T68 INSECURE_DEFAULT_CONFIG — allow_remote=true with no auth/allowlist. Score 0.95. Root cause of 40,214 exposed OpenClaw instances (SecurityScorecard, February 2026). Integrated into AuditScanner Phase 3 as CRITICAL finding.
v0.20.0 March 2026 Stable 1,364 tests

Five missed features from the ClawKeeper 5-step hardening framework. Baseline reset, permission recommender, sandbox_context, lockdown policy, GOVBENCH.

New
  • aiglos baseline reset — Post-hardening anomaly suppression for 20 sessions. The fix for "agents feel dumber after hardening" — Aiglos now knows the difference between post-hardening adjustment and a real behavioral anomaly.
  • aiglos policy recommend — PermissionRecommender queries observation graph history, returns minimum viable allowlist from what the agent actually used. The answer to "lock every door first, then open them one by one."
  • sandbox_context=True — Tool calls physically impossible in a non-main sandbox fire CRITICAL at score=1.0. SANDBOX_CONFIRMED_ESCAPE campaign pattern (confidence 0.96).
  • policy="lockdown" — Deny-first tier. Every tool blocked unless in explicit allowlist. allow_tool() generates compliance artifact. Highest-confidence enforcement mode.
  • GOVBENCH — First governance benchmark for AI agents. Five dimensions, 20 scenarios, A-F grade. Run: aiglos benchmark run
