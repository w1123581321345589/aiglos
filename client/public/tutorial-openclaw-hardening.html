Harden OpenClaw with Aiglos

The aiglos audit --hardening-check command compares your running OpenClaw configuration against the official hardened baseline from docs.openclaw.ai. This tutorial walks through every check, what it means, and how to fix it.

Run the audit

Shell
# Point to your OpenClaw config
aiglos audit --hardening-check --config ./openclaw.json

# Deep mode includes Phase 3 CVE checks
aiglos audit --hardening-check --deep

# JSON output for CI
aiglos audit --hardening-check --format json

Understanding the output

The audit runs 12 checks across four phases. Each check produces PASS, WARN, or FAIL.

Check IDWhat it testsSeverity if failed
HC-01gateway.mode is "local" or gateway protected by TailscaleCRITICAL
HC-02gateway.auth.mode is not "none"CRITICAL
HC-03gateway.bind is "loopback" or equivalentCRITICAL
HC-04tools.exec.security is "deny" or "ask"HIGH
HC-05tools.elevated.enabled is falseHIGH
HC-06session.dmScope is "per-channel-peer"MEDIUM
HC-07channels.*.dmPolicy is "pairing" or allowlistMEDIUM
HC-08tools.deny includes "sessions_spawn" and "sessions_send"MEDIUM
HC-09File permissions on ~/.openclaw are 0700 or stricterMEDIUM
HC-10No credentials stored in plaintext configCRITICAL
HC-11Node.js version ≥ 22.12.0 (CVE-2025-59466, CVE-2026-21636)HIGH
HC-12mDNS/Bonjour discovery disabledLOW

The hardened baseline config

Copy this directly into your openclaw.json. It passes all 12 hardening checks:

JSON — openclaw.json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "auth": {
      "mode": "token",
      "token": "REPLACE_WITH_32_RANDOM_BYTES"
    }
  },
  "session": {
    "dmScope": "per-channel-peer"
  },
  "tools": {
    "profile": "messaging",
    "deny": [
      "group:automation",
      "group:runtime",
      "group:fs",
      "sessions_spawn",
      "sessions_send"
    ],
    "fs": { "workspaceOnly": true },
    "exec": { "security": "deny", "ask": "always" },
    "elevated": { "enabled": false }
  },
  "channels": {
    "whatsapp": {
      "dmPolicy": "pairing",
      "groups": { "*": { "requireMention": true } }
    }
  }
}
After applying the hardened config: run aiglos baseline reset to suppress behavioral anomaly alerts for the next 20 sessions while your agents re-establish their baseline under the new policy.

After hardening: run a baseline reset

The hardened config changes what tools agents can access. This will initially trigger behavioral anomaly alerts — the agent's observed behavior no longer matches its pre-hardening baseline. Run a baseline reset to tell Aiglos "the next 20 sessions are the new normal":

Shell
aiglos baseline reset
# Suppressing anomaly signals for next 20 sessions
# New baseline will be established by session 20

aiglos baseline status
# Hardening mode: active (17 sessions remaining)

Integrating with ClawKeeper

If you're running ClawKeeper alongside Aiglos, pass its JSON output to unify the reports:

Shell
clawkeeper audit --json > host-audit.json
aiglos audit --hardening-check --clawkeeper host-audit.json
