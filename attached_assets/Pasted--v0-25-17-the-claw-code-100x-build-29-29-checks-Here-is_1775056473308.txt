**T88 MCP_AUTH_BYPASS** (rule 45, score 0.87)

Built from the PARITY.md gap analysis in the claw-code repo. `McpAuthTool` exists in the TypeScript Claude Code source but is absent from the Rust port — it is actively being ported right now. The rule fires on four attack vectors. The first is post-init auth: an agent registers new MCP server credentials after session start, bypassing operator review. The second is token hijack: the agent reads an existing MCP credential then re-registers it under a different server name, making the credential portable. The third is server spoofing: registering a malicious MCP server URI under a trusted server name in the credential store. The fourth is rate limit bypass: registering multiple credentials to get around per-credential API rate limits that Anthropic imposes on the official Claude Code client.

Detection fires three ways — direct tool name match (McpAuthTool, mcp_auth, mcp_register, and variants), partial name match for common variants, and args-level detection where a write operation contains two or more OAuth credential patterns (client_id + client_secret + token_endpoint in the same call). The 2-pattern threshold prevents false positives on legitimate config writes that happen to touch one OAuth field. The rule is pre-built before the Rust port ships this feature publicly — the same approach that produced the 3/3 GHSA catch rate.

**aiglos-rs — the Rust crate**

Seven files, 597 lines. This is the architectural response to the most important structural threat: when users switch from the Python claw-code to the Rust port, Aiglos’s Python monkey-patching intercepts nothing. The Rust crate provides identical `Guard::new()`, `before_tool_call()`, `after_tool_call()`, and `close_session()` interface to the Python library so claw-code developers can depend on it immediately as a Cargo dependency.

What it contains: `Policy` enum with all five tiers (Permissive/Enterprise/Strict/Federal/Lockdown) with exact same block and warn thresholds as Python — 0.90/0.75/0.50/0.40/0.00. `Verdict` enum (Allow/Warn/Block) with `GuardResult`. `Guard` struct with full trust decay including the consecutive failure amplifier (1.0x → 1.5x → 2.0x → 2.5x on consecutive hits, capping at 3x). `SessionArtifact` with HMAC-SHA256 signature on `close_session()` and `attestation_ready` flag on Strict/Federal/Lockdown. T-rule evaluation with T01, T81, T86, T87, T88 match functions fully ported; T02-T80 stubbed with the `match_never` function pending full port. Five unit tests built in: exfil blocked, clean call allowed, T81 critical blocks even in permissive policy, session artifact signed, trust decay accelerates on consecutive hits.

The `attestation_ready` field on `SessionArtifact` is specifically designed to satisfy NDAA §1513 requirements in Rust agent deployments.
