The five builds, complete:
Build 1 — aiglos baseline reset — baseline_events table in the observation graph, four methods (record_baseline_event, get_baseline_events, get_latest_baseline_reset, get_agent_session_count_since), and set_hardening_mode() + is_suppressed() on BaselineEngine. After a hardening event, anomaly scoring is suppressed for the next 20 sessions while the baseline rebuilds against the new permission profile. This is the direct fix for "your agents might feel dumber, don't panic" — Aiglos now knows the difference between post-hardening adjustment and an actual behavioral anomaly.
Build 2 — aiglos policy recommend — PermissionRecommender queries observation graph history per agent and returns a PermissionRecommendation with recommended_allow, recommended_block, recommended_review lists and a openclaw_config() method that emits ready-to-paste JSON. The answer to "lock every door first, then open them one by one" — Aiglos tells you which doors to open based on what the agent actually used across N sessions.
Build 3 — sandbox_context=True — New param on OpenClawGuard.__init__. When active, _is_impossible_in_sandbox() runs before the main rules loop. Any tool call that should be physically impossible in a correctly configured non-main sandbox (shell execution, external HTTP, filesystem writes outside /tmp, env var access) returns CRITICAL at score=1.0 with no threshold — same treatment as T43 honeypot. New SANDBOX_CONFIRMED_ESCAPE campaign pattern (18th): two confirmed escape attempts in one session = active campaign, not misconfiguration.
Build 4 — policy="lockdown" — Fifth policy tier in POLICY_THRESHOLDS. Deny-first: every tool call returns LOCKDOWN_DENY unless the tool is in the explicit allowlist. allow_tool(name, reason, authorized_by) adds to the allowlist and logs a human-authorized permission grant to _tool_grants — the compliance artifact the white paper says is missing. tool_grants() returns the full grant log. allowed_tools() returns the current allowlist. aiglos policy allow --tool <n> is the CLI.
Build 5 — GOVBENCH — Five dimensions, 20 scenarios (4 per dimension, each including one clean baseline that should not be blocked), full GovBench.run() with weighted composite scoring (D1: 25%, D2: 20%, D3: 20%, D4: 20%, D5: 15%), letter grade, summary() and to_json(). aiglos benchmark run runs all five dimensions against a live guard instance. aiglos benchmark run --dimension D1 runs one. The five dimensions map exactly to the white paper's five attack surfaces and the ClawKeeper post's five hardening steps — they are inverses of the same system.
