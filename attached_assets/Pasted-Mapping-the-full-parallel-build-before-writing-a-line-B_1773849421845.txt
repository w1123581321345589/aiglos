**What was already built (v0.23.0 was ahead of where we left off):**

**T70 `ENV_PATH_HIJACK` (score 0.88, critical)** — fires on PATH, LD_PRELOAD, LD_LIBRARY_PATH, PYTHONPATH, and NODE_PATH modification in execution contexts. GHSA-mc68-q9jw-2h3v is the empirical proof of concept. The rule closes the gap that T03 left: T03 catches shell metacharacter injection, T70 catches the upstream environment manipulation that redirects execution before the shell even sees the command.

**`ghsa_watcher.py`** — 629 lines across three classes. `AdvisoryMatch` holds the coverage result for one advisory. `PendingRule` is generated when an advisory has no matching rule — written to `pending_rules/GHSA-xxxx.json` for human review. `GHSAWatcher` pulls from four sources: GitHub Advisory API, OSV.dev, NVD CVE feed, and the jgamblin/OpenClawCVEs GitHub feed (the same feed powering ClawKeeper's security page). `TAXONOMY_KEYWORDS` maps every T-rule ID to the natural language keywords that appear in vulnerability descriptions for that attack class. `run_once()` cycles through all sources, seeds known advisories, matches new findings against the taxonomy, and writes pending rules for gaps. `coverage_report()` outputs the formatted table. CLI commands: `aiglos autoresearch watch-ghsa`, `aiglos autoresearch ghsa-coverage`, `aiglos autoresearch ghsa-rematch`.
