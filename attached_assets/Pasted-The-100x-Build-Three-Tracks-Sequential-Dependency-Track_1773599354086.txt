Track A — Behavioral Baseline Engine
What it is: Per-agent statistical fingerprint built from the observation graph. After 20+ sessions, Aiglos knows what normal looks like for this specific agent. Deviation from that baseline is an independent, orthogonal signal.
What data already exists: The events table has agent_name (via sessions), rule_id, surface, verdict, tier, latency_ms, and timestamp per event. The sessions table has total_events, blocked_events per session. Everything needed to compute a baseline is already being stored.
New files:

aiglos/core/behavioral_baseline.py — AgentBaseline dataclass, BaselineEngine that computes rolling distributions across three feature spaces:

Rule frequency distribution — how often does this agent trigger each rule family? Expressed as a probability vector over T01-T39. Anomaly = KL-divergence from historical mean.
Session event rate — typical events-per-session, blocked-per-session, warn rate. Anomaly = z-score against rolling 30-session window.
Surface mix — ratio of MCP:HTTP:subprocess events. A coding agent with a sudden shift toward HTTP-heavy traffic is anomalous even if no individual call triggers a rule.


aiglos/adaptive/observation.py update — agent_baselines table, upsert_baseline(), get_baseline(), compute_baseline_from_history(agent_name, n_sessions=30)
aiglos/adaptive/inspect.py update — 11th trigger: BEHAVIORAL_ANOMALY — fires when current session deviates from agent baseline by >2σ across any of the three feature spaces. Severity calibrated by number of σ. Amendment candidate: True (deviation might be legitimate new behavior).
aiglos/integrations/openclaw.py update — enable_behavioral_baseline() method, baseline check at close_session() — scores the session against its own historical norm and includes baseline_deviation section in artifact.

Tests: tests/test_behavioral_baseline.py — baseline computation, anomaly detection, z-score calibration, artifact section structure.
What this unlocks: The false positive death spiral breaks. A DevOps agent reading credentials is normal; the baseline says so and the alert doesn't fire. A code agent doing the same is anomalous; the baseline catches it even if the per-call rule scores borderline.
