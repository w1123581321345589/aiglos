11th campaign pattern. Sequence: T36_AGENTDEF write followed by T01 or T27 injection in the same session. Confidence 0.91. Build 3 — SourceReputationGraph (source_reputation.py)
Context Hub mechanism applied to threat intelligence. SourceReputationGraph maps URLs, domains, and document fingerprints to their injection history across sessions. Four levels: CLEAN, SUSPICIOUS (1–2 events, 0.75x threshold), HIGH_RISK (3+ events or max score ≥ 0.75, 0.5x threshold), BLOCKED (8+ local events or 5+ network flags). Domain propagation: a flag on api.attacker.com/specific-path applies a dampened score to the entire api.attacker.com domain.
record_event() called by after_tool_call() whenever the injection scanner scores ≥ 0.25. get_risk() called by before_tool_call() before any fetch operation — if BLOCKED, returns immediately without hitting the network. get_shareable_reputation() returns anonymized domain-level scores for federation sharing.
source_records table in the observation graph. upsert_source_record() preserves MAX score on conflict so a stale low-score update can never downgrade a high-risk record. list_source_records() for the CLI.
enable_source_reputation() on OpenClawGuard. enable_source_reputation=True kwarg wired into attach().
aiglos reputation CLI — top, show, domains, shareable. Color-coded by level.
The compounding: session 1 an endpoint returns injection attempts, annotated. Session 2 that endpoint gets elevated scrutiny before content even loads. Session N the source is blocked. The goldfish problem is solved at the source level, not just the content level.
