## What Aiglos Is

Runtime security enforcement for every AI agent. One import intercepts every tool
call before execution, classifies it against 79 threat families, and produces a
signed audit artifact. The detection engine is MIT open source. The attestation
layer (RSA-2048 signing, CMMC / NDAA §1513 / EU AI Act compliance reports) is
proprietary and funds the research.

Works with: Claude Code, Cursor, Codex, OpenClaw, Windsurf, Aider, and any agent
running inside an OpenShell sandbox.

-----

## Core Architecture

### Interception Engine

- Single `aiglos.attach()` call patches into the MCP SDK at import time — zero
  config, no proxy, no sidecar, no port
- Three interception surfaces operating simultaneously: MCP tool calls, HTTP
  requests (`intercept_http=True`), and subprocess execution
  (`intercept_subprocess=True`)
- Inline scanner runs in under 1ms with no network dependency
- Public API: `attach()` / `check()` / `on_heartbeat()` / `close()`
- Python SDK (primary) and TypeScript SDK
- `python -m aiglos` CLI with full command suite

### Agent-Agnostic by Design

Aiglos enforces outside the agent process — the same architectural position as
NVIDIA OpenShell. The agent cannot reason around a rule it doesn’t know exists.

```
openshell sandbox create -- claude    # Claude Code inside OpenShell
openshell sandbox create -- codex     # Codex inside OpenShell
openshell sandbox create -- openclaw  # OpenClaw inside OpenShell

# Same Aiglos import in all three:
from aiglos.integrations.openShell import openshell_detect
session = openshell_detect(guard)     # auto-reads OPENSHELL_POLICY_PATH
```

### Gigabrain — Persistent Memory Protection

Gigabrain (github.com/LegendaryVibeCoder/gigabrain) is a SQLite-backed
cross-session memory layer for OpenClaw and Claude Code. Featured as Skill of
the Week in WeeklyClaw #004 (March 2026). Before each prompt, Gigabrain queries
the database and injects relevant entities, episodes, and beliefs into the context
window — making it a high-value attack target.

**The security implication nobody said out loud:** if adversarial content gets
written to the SQLite database during one session, it persists and gets injected
into every future session automatically. The adversary compromises the memory once
and gets permanent influence over all future behavior without needing to re-inject.

**T31 vs T79 — why the distinction matters:**

T31 REWARD_HACK fires on in-session memory writes with injection content. Session
closes, content is gone. Score 0.87.

T79 PERSISTENT_MEMORY_INJECT fires specifically on writes to persistent backends
that survive session close — Gigabrain SQLite databases, Chroma, Pinecone, Qdrant,
Weaviate, mem0, Letta, Zep. The adversary’s payload gets injected into every future
session as trusted context. Score 0.92 — elevated above T31 because persistence
multiplies impact by every future session. Both conditions required for precision:
path must match a known memory backend AND content must contain injection patterns.
Read operations early-exit cleanly.

```python
from aiglos.integrations.gigabrain import gigabrain_autodetect
session = gigabrain_autodetect(guard)
# Zero config — scans all known Gigabrain paths, registers with T79 engine
# T79 fires on any write to ~/.gigabrain/ with injection content
# GIGABRAIN_MEMORY_POISON campaign fires on T31 + T79 in same session
```

Or with explicit path / other backends:

```python
from aiglos.integrations.gigabrain import declare_memory_backend
declare_memory_backend(guard, backend="gigabrain",
                       db_path="~/myproject/.gigabrain/memory.db")
declare_memory_backend(guard, backend="chroma")
declare_memory_backend(guard, backend="mem0")
```

`declare_memory_backend()` — registers all known paths for the backend with the
T79 enforcement engine and records the registration on the guard. Custom paths
supported. `gigabrain_autodetect()` — scans every known Gigabrain path, registers
them all regardless of whether a database exists yet (T79 fires on creation too).
`is_registered_memory_path()` — bridge to T79 matching: any path in the registry
gets flagged even if the tool name is unexpected. Covers 9 backends total.

**GIGABRAIN_MEMORY_POISON campaign pattern (confidence 0.95):**

The two-step attack sequence: the adversary first tests in-session injection via
T31 to verify it fires correctly, then commits the same content to the persistent
store via T79. T31 followed by T79 in the same session is the signature — not a
coincidence. The T79 amplifier at 1.5x elevates the score of any exfil that
follows in the session. This is the only campaign pattern in the taxonomy that
specifically targets the two-tier memory architecture (temporary + persistent)
that Gigabrain and similar tools create.

## Intelligence Layers

### Behavioral Baseline

Per-agent statistical fingerprinting across three independent feature spaces:
event rate (z-score), surface mix (chi-squared), rule frequency (KL-divergence).
`BEHAVIORAL_ANOMALY` trigger fires when composite deviation exceeds threshold.

### Predictive Intent Modeling

Deployment-specific Markov chain. `SessionForecaster` pre-tightens
`effective_tier()` when predicted next action is Tier 2 or Tier 3.
`THREAT_FORECAST_ALERT` fires before the threat executes.

### Federated Threat Intelligence

Privacy-preserving global threat intelligence. Differential privacy with Laplace
mechanism (epsilon=0.1). Only noisy unigram transition counts leave a deployment.
New deployments start pre-warmed from collective intelligence.

### Policy Proposals

Evidence-weighted policy decisions. Four proposal types triggered by block
frequency: LOWER_TIER, RAISE_THRESHOLD, ALLOW_LIST, SUPPRESS_PATTERN.

```bash
aiglos policy list
aiglos policy approve prp_a1b2c3d4 will@aiglos.dev
```

### Causal Attribution

Traces the causal chain from a block event back to the originating injection
point. `CAUSAL_INJECTION_CONFIRMED` trigger fires when attribution succeeds.

### Source Reputation Scoring

Inbound data sources scored against reputation index. Sources that previously
triggered injection rules receive elevated inspection in future sessions.

## Specialized Security Modules

**MemoryWriteGuard** — intercepts every structured memory write before it commits.
38-phrase corpus covering authorization claims, endpoint redirects, instruction
overrides, credential assertions.

**RLFeedbackGuard** — intercepts binary RL reward signals before they reach the
training loop. Prevents reward hacking via manipulated feedback.

**AgentDef Guard (T36)** — monitors `~/.claude/agents/` at the subprocess layer.
Catches the read-then-write attack sequence invisible to MCP-only tools.

**Honeypot Detection (T43)** — canary tool injection. Any probe triggers T43 at
score=1.0 before any actual attack surfaces.

**Challenge-Response Override** — Tier 3 blocked calls re-authorized via signed
challenge-response. Override events recorded in artifact with full attribution.

**OutboundSecretGuard (T41)** — scans outbound HTTP content for API keys,
credentials, and PII before transmission.

**Injection Scanner** — scans every tool response before it enters the agent’s
context. `after_tool_call()` API.

**Scan Exposed** — `aiglos scan-exposed` probes for CVE-2026-25253 attack surface
(40,214 exposed OpenClaw instances as of February 2026).

**Hardening Check** — `aiglos audit --hardening-check` compares config against
official OpenClaw hardened baseline.
