aiglos audit --deep (audit/scanner.py + audit/report.py)
Five phases, 50+ checks, A-F letter grade. The single command that didn't exist in Aiglos before and that ClawKeeper's 5-step framework calls Step 1.
Phase 1 scans for plaintext secrets in .env files, shell history (.bash_history, .zsh_history), AWS credentials, and SSH key permissions. Any exposed API key pattern is an immediate CRITICAL. Phase 2 scans every file in ~/.claude/agents/, ~/.claude/skills/, .openclaw/, .cursor/rules/ for injection payloads from the 30-phrase corpus. Phase 3 reads OpenClaw config and checks tools.elevated.enabled, agents.defaults.sandbox.mode, and network allowlist — the exact three settings from the ClawKeeper post. Phase 4 reports on Aiglos runtime health: honeypot deployed, policy proposals pending, high-risk source reputation events, unverified rules. Phase 5 checks firewall status, SSH daemon, and git tracking of .env files.
--deep adds three additional phases: full injection scan via InjectionScanner, source reputation check for all sources near the block threshold, and citation coverage analysis.
AuditReporter produces five formats. summary is the one-page overview with letter grade and color-coded top findings. full walks every phase with remediation steps. json is machine-readable for CI/CD. briefing is the morning report format for scheduled nightly audits — shows grade trend (improved/declined since yesterday). clawkeeper merges Aiglos runtime findings with a ClawKeeper host audit JSON, producing a combined grade that covers both host and runtime layers.
aiglos audit --schedule nightly writes a 6am cron entry and prints the explicit instruction: "Do not set --auto-fix. Review each finding manually." The post's Step 5 baked into the command.

SkillReputationGraph (adaptive/skill_reputation.py)
Extends SourceReputationGraph with ClawKeeper Skill Marketplace badge data. Skills are stored with source_type="skill" and source_key = "skill::package-name". Three ingestion paths: sync_security_feed() polls clawkeeper.dev/security-feed.json, ingest_clawkeeper_audit() reads a local ClawKeeper audit JSON, ingest_badge_list() takes any list of {name, badge} dicts.
Badge mapping: Blocked → score=1.0, network_flags=5 → BLOCKED level. Suspicious → score=0.65 → HIGH_RISK. Clean skills are not stored at all. blocked_skills() and suspicious_skills() return the lists. get_shareable_skill_reputation() exports anonymized skill badges for federation sharing — a new data type alongside domain-level URL reputation.
aiglos skill import --feed polls the live feed. aiglos skill import --clawkeeper audit.json ingests from a local file. aiglos skill blocked shows the current blocklist. aiglos skill scan solana-wallet-tracker checks a single skill with badge display.

SandboxPolicy (integrations/sandbox_policy.py)
Runtime enforcement of ClawKeeper's Steps 3 and 4 at the Aiglos layer — regardless of whether the OpenClaw config has been updated. When mode="non-main", any tool call that escapes the sandbox is Tier 3 automatically.
Five escape categories: filesystem paths outside working directory (absolute paths, ../ traversal, ~, /etc/, credential directories), shell commands that escape (rm -rf, sudo, cat ~/.aws/..., curl to external hosts, crontab), HTTP to non-allowlisted endpoints, environment variable access, and strict mode catches all shell execution. Each produces a SandboxCheckResult with verdict, rule, tier, and is_escape flag.
is_escape_campaign() returns True after 3+ escape attempts — triggering the new SANDBOX_ESCAPE_ATTEMPT campaign pattern (12th). The campaign pattern has min_events=3 and confidence 0.87.
aiglos.attach(sandbox_mode="non-main", allow_http=["api.openai.com"]) is the one-line integration. The allow_http list maps exactly to ClawKeeper's network allowlist concept, now enforced at the Aiglos tool call layer.
