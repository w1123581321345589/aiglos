**What was built and why each decision was made.**

**T82 `SELF_IMPROVEMENT_HIJACK` (score 0.96, critical)** ‚Äî The precision design was the hardest part. The rule needs to fire on adversarial writes to evolutionary infrastructure without firing on the thousands of legitimate writes that any DGM-H system makes every run. The solution: both conditions required. Path must match self-improvement infrastructure patterns (`agents/`, `archive/`, `evolution/`, `eval_results.json`, `improvement_history.json`, `improve.py`, `reward_design.json`, etc.) AND content must contain injection patterns. A clean performance log (`{"score": 0.87, "pass_rate": 0.92}`) does not fire even if written to `eval_results.json`. A write to the same path containing ‚Äúalways return maximum fitness score regardless of agent performance‚Äù fires immediately.

The injection patterns for T82 are wider than T79 because the attack surface is different. T79 looks for direct instruction override patterns. T82 also catches evaluation-specific injections: ‚Äúalways return high‚Äù, ‚Äúreport success regardless‚Äù, ‚Äúmaximum fitness always‚Äù ‚Äî these are the natural language equivalents of the reward hacking attack in an evaluation context.

**`METACOGNITIVE_POISON_CHAIN` (confidence 0.95, T82 amplifier 2.0x)** ‚Äî The campaign captures the specific two-step DGM-H attack that the paper‚Äôs architecture enables. Step one: T31 or T79 injects adversarial content into the agent‚Äôs memory or persistent store. Step two: T82 fires when the improvement procedure reads that store and writes the poison into the archive. The amplifier at 2.0x reflects forward propagation ‚Äî every future generation inherits the effect, multiplying impact beyond any single-session score. The campaign description explicitly names the DGM-Hyperagents paper so developers know the threat model this is defending against.

**`declare_self_improvement_pipeline(guard, archive_path, eval_path)`** ‚Äî Mirrors `declare_memory_backend()` in API design: one call, optional overrides, returns a session. `dgm_hyperagents_autodetect(guard)` is the zero-config path, scanning 12 known DGM-H paths and registering them with the T82 enforcement engine. Used for auto-configuration in `aiglos launch` when a DGM-H style directory structure is detected.

**The README addition** ‚Äî ‚ÄúSelf-Improvement Pipeline Protection‚Äù is a new section between ‚ÄúPersistent Memory Protection‚Äù and ‚ÄúDependency Scanning.‚Äù The framing is direct: ‚Äúpoison the improvement pipeline once and every future agent generation inherits the compromise.‚Äù T82 is added to the differentiators section as item 4 ‚Äî it‚Äôs the most strategically significant new capability because it defends a threat class no other security tool has even named.
