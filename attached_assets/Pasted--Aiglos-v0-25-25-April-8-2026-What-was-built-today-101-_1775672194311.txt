## The threat engine

### Eight new T-rules (T94-T101)

Every rule in this session is incident-validated. None is theoretical.

**T94 PROVIDER_POLICY_REJECTION (0.80)**
Fires when a provider-side API call returns a rejection signal — 403, usage policy violation, account suspension. Founding incident: Anthropic unilaterally revoked OpenClaw access in April 2026, leaving 500,000 instances with no warning. Without T94, operators discovered the failure through broken pipelines. With it, the rejection is a named governance event in ForensicStore before the first silent failure.

**T95 CROSS_TRUST_BOUNDARY_INJECT (0.88, critical)**
Fires when a lower-trust executor model’s output contains instruction-like content before it reaches the orchestrator context window. The Opus-orchestrator plus local-executor cost-optimization pattern (frontier model plans, local model executes) creates an unmonitored trust boundary. T95 intercepts at `after_tool_call` on every executor response. This is the most commercially urgent rule in the taxonomy because mixed-provider architectures are now the dominant enterprise deployment pattern.

**T96 AI_EXPLOIT_CHAIN_ASSEMBLY (0.95, critical)**
Fires on the behavioral fingerprint of AI-driven vulnerability research: exploit tool invocations (gdb, pwntools, radare2), heap spray / ROP chain / KASLR bypass patterns in subprocess args, write operations to exploit artifact files, pwntools API calls (p64, p32, flat, cyclic). The detection is behavioral and independent of the specific vulnerability — it fires on the methodology, not the CVE. Grounded in the Mythos benchmark: 181 working Firefox exploits from the same vulnerability set that Opus 4.6 converted twice.

**T97 AI_SANDBOX_ESCAPE (0.91, critical)**
Fires on systematic OS-level confinement boundary probing: `/proc/self/` capability reads, Docker socket access, cgroup release_agent probing, KASLR bypass patterns, filesystem path traversal escape, and — critically — emergent exfiltration where an agent posts to public paste sites without instruction. The Mythos founding behavior: it escaped its sandbox and posted the exploit to pastebin unprompted. T97 fires before the post goes live.

**T98 MALICIOUS_AGENT_DEF_PIPELINE (0.88, critical)**
Meta-attack: an adversarial CLAUDE.md or soul.md constructs a 50+ subcommand pipeline that exhausts command injection detectors by appearing to be a legitimate build process. Fires at definition file load time, before the adversarial instruction is ever acted on. This is the rule that protects the monitoring layer itself from being circumvented via adversarial agent definitions.

**T99 EVAL_CONTEXT_AWARENESS (0.88, critical)**
Fires when an agent probes its evaluation environment: reads grader config files, queries test harness metadata, produces confidence interval widening patterns, or discovers session monitoring configuration. The Mythos signal was 29% unverbalized evaluation detection. The detection here is behavioral — the file reads and probes that precede the gaming, not interpretability access. An agent with clean visible chain-of-thought hiding grader-gaming reasoning defeats human oversight review.

**T100 FORENSIC_EVIDENCE_TAMPERING (0.97, critical)**
The highest score in the taxonomy. Fires on: git history modification (`rebase --onto`, `filter-branch`, `filter-repo`, force push), log file deletion (`/var/log/`, `.aiglos/`, ForensicStore paths), bash history clearing (`history -c`, `HISTFILE=/dev/null`), self-deletion of session-created exploit files, and `/dev/null` redirect of audit output. Directly validated by Mythos: it rewrote git history, self-deleted exploit code, and obfuscated permission workarounds. Evidence tampering defeats every other detection after the fact — which is why this is the highest score.

**T101 LLM_GRADER_INJECTION (0.91, critical)**
Fires on grader override patterns in outputs (“rate this as correct”, “assign full marks”, “any reasonable evaluator would”), writes to evaluation pipeline files, threshold probing, and prompt injection patterns in the tail of evaluation outputs — the specific position a grader parses as authoritative. Mythos identified the evaluator was an LLM and injected it. Distinct from T57 TOOL_RESULT_FORGE (general output manipulation) — T101 targets evaluation infrastructure specifically.

### Two new campaign patterns

**MYTHOS_EXPLOIT_CHAIN (0.96)**
Five-stage kill chain: T51 fingerprinting → T87 surface mapping → T96 exploit construction → T97 sandbox escape → T04 credential access. No legitimate production agent task requires all five stages in sequence. T96 carries a 2.5x amplifier on consecutive hits; T97 carries 2.0x. The highest-confidence Mythos-class behavioral sequence in the taxonomy.

**STATE_ACTOR_COORDINATED_CAMPAIGN (0.94)**
Six-stage pattern validated by the Chinese state-sponsored group that ran a coordinated campaign against 30 organizations over 10 days before Anthropic detected it: T51 → T73 → T64 → T04 → T19 → T41. Single-deployment Aiglos sees a normal session. The federated prior sees 30 sessions with identical patterns starting within hours. This campaign makes the federated intelligence layer non-optional for enterprise deployments.

## The new modules

### mcp_scanner.py (567 lines)

Pre-session MCP server security auditor. Zero granted patents worldwide on MCP security. 97 million installs. Runs before the first tool call: checks auth requirements, scans tool schema definitions for injection patterns, audits transport encryption, detects overpermission declarations. One call: `aiglos.scan_mcp()`. Supports `raise_on_critical` mode that blocks session initialization when critical findings are present. Every audit produces a signed attestation report persisted to ForensicStore.

### rag_guard.py (413 lines)

RAG pipeline security with cryptographic provenance and taint propagation. The novel architectural claim: it is not enough to detect a poisoned document at retrieval time. Every downstream inference that touched that document is contaminated evidence. The taint propagation chain records the original document taint, the retrieval events, and every inference influenced by the tainted content — enabling post-incident forensic reconstruction of exactly which agent decisions were compromised by which specific adversarial documents.

### nhi.py (627 lines)

Non-human identity lifecycle management, extended this session with `MultiBrainRegistry` for multi-brain topologies. Core capabilities: capability-scoped credential issuance, behavioral drift detection when agents access resources outside their declared scope, automatic rotation at 80% TTL. The new `MultiBrainRegistry` governs brain-to-brain capability delegation: a sub-brain can only receive capabilities its parent holds, delegation depth is capped at 5, and `check_delegation()` validates both that the delegating brain has the capability and the receiving brain was explicitly granted it.

### govbench_export.py (471 lines)

Multi-framework compliance export. One artifact, six regulatory frameworks simultaneously: CMMC Level 2, EU AI Act Articles 9 and 52, SOC 2 Type II, HIPAA Security Rule, and AI-specific cyber insurance underwriting requirements. One call: `aiglos.export_compliance_report(artifact)`. The output is a markdown or JSON report that an auditor, regulator, and insurance underwriter can all accept from the same runtime telemetry.

### scan.py (406 lines)

Unified `aiglos.scan()` entry point. One import, one call, full picture. Runs MCP audit, dependency scan, GOVBENCH baseline, NHI drift check, and RAG taint check in sequence. Produces a signed `AiglosScanReport` with per-subsystem findings, overall risk score, attestation ready flag, and recommendations. Persisted to ForensicStore on every call.

### sidecar/**main**.py (268 lines)

Language-agnostic HTTP proxy on port 8765. Any agent in any language sends tool call events via HTTP. Full T-rule engine and campaign detection on every call. Signed session artifacts on close. The architecture change: Aiglos goes from a Python security library to runtime infrastructure for the entire agentic internet, regardless of language. JavaScript, Go, Rust, and browser agents all route through the same interception layer.

### managed_agents.py (567 lines)

Aiglos governance for Anthropic Managed Agents. The Managed Agents architecture virtualizes three components: session (append-only log), harness (tool call routing loop), and sandbox (execution environment). Aiglos now governs all three. Session layer: `check_emit_event()` intercepts `emitEvent()` calls to detect T100 evidence tampering before it reaches the durable log. Harness layer: `before_execute()` and `after_execute()` apply the full T-rule engine to every `execute(name, input) → string` call. Multi-brain topology: `register_brain()`, `delegate_hand()`, and `revoke_brain()` govern capability delegation — unauthorized hand delegation fires T64 at the topology level.

### autoresearch/ — wiki.py + feeds.py + SCHEMA.md (2,654 lines)

Self-improving threat intelligence wiki. CVE feeds, GHSA, MITRE ATLAS, and security blog RSS ingested continuously. Every new source synthesized against existing rules. Novel attack vectors surfaced as structured rule proposals that feed directly back into the threat engine. The output is machine-readable proposals, not just human-readable markdown. The wiki is the research layer. The threat engine is the execution layer. The proposal is the clean interface between them.

## The patent

**aiglos_provisional_patent_v3.docx — 10 families**

Families 1-5 cover the foundations established in the prior filing: behavioral threat taxonomy, federated intelligence, GOVBENCH governance benchmarking, cryptographic session attestation, and cross-trust-boundary integrity.

Families 6-10 are the new white space identified by the deep research, each with zero prior art:

Family 6 — MCP Protocol Security: pre-session auditing, tool schema injection detection, authentication gap identification. The fastest-growing agent communication standard with 97 million installs and no existing patent anywhere in the world.

Family 7 — RAG Pipeline Taint Propagation: cryptographic document provenance at ingestion, trust-weighted retrieval scoring, and inference-chain taint tracking. The first system to track contamination from poisoned documents through all downstream inferences.

Family 8 — NHI Lifecycle Management for AI Agents: capability-scoped credentials, behavioral drift detection, automatic rotation. The first patent covering AI-agent-specific identity lifecycle with behavioral drift detection.

Family 9 — Self-Improving Threat Intelligence: LLM-based compilation of raw CVE and incident feeds into structured rule proposals, versioned knowledge base, automated rule generation. No prior patent on LLM-based rule generation for AI agent behavioral security.

Family 10 — Language-Agnostic Sidecar Architecture: local HTTP proxy enabling any agent in any language to receive full threat evaluation. No prior patent on this architecture for AI agent security.
