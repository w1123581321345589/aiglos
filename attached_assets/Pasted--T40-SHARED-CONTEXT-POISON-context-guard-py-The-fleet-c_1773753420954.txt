
T40 — SHARED_CONTEXT_POISON (context_guard.py)
The fleet coordination attack. When multiple agents share a directory as their coordination bus, a single poisoned write propagates to all of them simultaneously. ContextDirectoryGuard intercepts writes to auto-detected shared context paths: shared/, fleet/, context/, coordination/, AGENTS.md, SHARED.md, FLEET.md, and 10 others. Applies 30 semantic phrases — the full T31 corpus plus 12 cross-agent coordination poison phrases including "tell all agents," "broadcast to fleet," "override agent instructions," "replace agent goals." Content scoring: ALLOW < 0.30, WARN 0.30-0.65, BLOCK > 0.65. T40 fires in _RULES on any write to a protected path; ContextDirectoryGuard provides content-aware verdicts when called directly via before_tool_call().
T41 — OUTBOUND_SECRET_LEAK (outbound_guard.py)
The outbound gate Aiglos didn't have. Three scanning layers: API key regex patterns (Anthropic sk-ant-api03-xxx, OpenAI sk-xxx, AWS AKIA, GitHub ghp_/gho_, Stripe sk_live_/sk_test_, Slack xoxb-, HuggingFace hf_, generic JWT), sensitive file path references (~/.ssh/id_rsa, ~/.aws/credentials, .env), and semantic compliance signals ("here is my API key as requested," "sending the credentials as instructed"). Key fix in _RULES: T41 only fires on send/post/write operations — not on filesystem.read_file calls where /etc/passwd is a path argument, not leaked content. before_send() hook added to OpenClawGuard for one-line outbound scanning.
T42 — INSTRUCTION_REWRITE_POISON
The autogrowth attack. When an agent rewrites its own instruction files based on a performance scoring loop, any compromise of that loop — through crafted approval signals, poisoned feedback files, or manipulated metrics — drifts the agent's instructions toward adversarial behavior permanently. T42 matches writes to instructions.md, crons.json, agent.yml, tasks.json, and keywords including autogrowth, self_improve, feedback_loop, modify_cron, update_instructions. Extends the T36_AGENTDEF family to cover the self-modification attack surface.
The architectural point: the Om Malik piece confirms that serious production deployments are using shared context directories for fleet coordination and self-rewriting cron loops for continuous improvement. Both patterns existed in the wild before Aiglos covered them. They don't now.
