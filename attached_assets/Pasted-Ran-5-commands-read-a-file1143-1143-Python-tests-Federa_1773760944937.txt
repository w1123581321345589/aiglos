Federation Server (server/)
Five files, one Railway deploy. The prior was an island before this existed.
aggregator.py implements federated averaging with two privacy layers. The client applies Laplace noise at epsilon=0.1 before submitting. The server applies Gaussian sigma=0.5 before broadcasting. What gets stored: aggregated noisy counts, total contribution integer, last timestamp. Nothing traceable to a deployment. The S-curve local weight schedule means a new deployment starts at 20% local weight and reaches 80% at 100 sessions — the global prior dominates early and yields to local evidence as the deployment matures.
main.py is four routes: GET /v1/health (unauthenticated liveness), GET /v1/status (public stats), GET /v1/prior (free + pro), POST /v1/contribute (pro only). FastAPI with CORS, lifespan-based prior loading from Supabase on startup so a server restart doesn't reset the prior to zero.
auth.py validates ak_free_xxx and ak_live_xxx key formats. Free tier pulls. Pro tier contributes and pulls. AIGLOS_MASTER_KEY env var bypasses for admin operations. The Supabase validation hook is stubbed out with a clear TODO — the MVP accepts all well-formed keys, which is the right call for getting the network going fast.
store.py has two implementations: _MemoryStore (in-process, no dependencies, works in development and CI) and the Supabase client path that activates when SUPABASE_URL and SUPABASE_SERVICE_KEY are set. The schema SQL is in README.md — three tables, copy-paste into the Supabase SQL editor.
Dockerfile + railway.json — railway up from the repo root deploys the server in under two minutes. The health check path is wired so Railway restarts the container if the server stops responding.

TypeScript SDK (sdk/typescript/src/)
Everything from v0.11.0 through v0.17.0 is now available in TypeScript. Every feature that existed only in Python until today.
behavioral_baseline.ts — BaselineEngine with the same three-component scoring: event rate z-score, surface mix chi-squared, rule frequency KL-divergence. Same weights (40/30/30), same thresholds, same MIN_SESSIONS=5 cold start. record(stats) and score(current) return BaselineScore with composite, component breakdown, risk level, and narrative.
policy_proposals.ts — PolicyProposalEngine with all four proposal types and the same confidence formula. recordBlock() returns a PolicyProposal | null. approve() and reject() update status. The same evidence tier thresholds as Python: 5 blocks for LOWER_TIER, 8 for RAISE_THRESHOLD, 10 for ALLOW_LIST, 15 for SUPPRESS_PATTERN.
federation.ts — FederationClient with Laplace noise via inverse CDF, rate limiting (600ms between pushes), 8s timeout with AbortController, graceful degradation on all network errors. pullPrior(), pushTransitions(), and mergeIntoPrior() with the same S-curve local weight schedule. Hits the live https://intel.aiglos.dev endpoint by default.
security_surfaces.ts — all five new surfaces in one file: ContextDirectoryGuard (T40, same protected path sets, same 30-phrase corpus), OutboundSecretGuard (T41, same 8 API key regexes, path patterns, semantic compliance patterns), HoneypotManager (T43, same 8 default files, checkToolCall fires before any scoring), OverrideManager (6-char challenge codes, 120s expiry, 3 attempts, prints the terminal box to console), SourceReputationGraph (URL/domain normalization via new URL(), same 4 reputation levels, same decay logic).
index.ts — full rewrite. attach() accepts the extended AiglosFullConfig and wires all layers. beforeToolCall() runs T43 → T40 → source reputation in the correct priority order. afterToolCall() updates source reputation from injection scan results. beforeSend() delegates to OutboundSecretGuard. requestOverride() and confirmOverride() are first-class exports. The status() function reports which intelligence layers are active.
