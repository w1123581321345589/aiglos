**T68 `INSECURE_DEFAULT_CONFIG`** ‚Äî fires when `allow_remote=true` appears in any tool call or config write alongside disabled authentication or no auth keywords at all. Score 0.95, critical. The exact misconfiguration behind all 40,214 exposed instances. The match function has three tiers: `allow_remote=true` + explicit auth disabled fires, `allow_remote=true` + no auth keywords at all fires, `allow_remote=true` + auth credentials present passes. Integrated into AuditScanner Phase 3 as a CRITICAL finding that drops the security grade immediately to D or F.

**`sandbox_type` extension** ‚Äî `_SANDBOX_CONSTRAINTS` dict defines three environment profiles. Firecracker adds `system_paths_readonly=True` ‚Äî writes to `/etc/`, `/usr/`, `/bin/`, `/sbin/`, `/lib/` are physically impossible in a Firecracker VM whose root image is immutable. A write to `/etc/cron.d/` inside a Firecracker sandbox is not suspicious ‚Äî it is confirmed escape. Docker has its own profile: env vars can be passed at launch, network depends on run flags. `aiglos.attach(sandbox_context=True, sandbox_type="firecracker")` gets you the KiloClaw-specific constraint table automatically.
