## What Was Built This Session

### Five New Threat Rules (T94-T98)

#### T94: PROVIDER_POLICY_REJECTION

**Score:** 0.80 | **Critical:** No

Fires when an agent’s LLM API call returns a provider-side policy rejection signal — 403 errors, usage policy violations, account suspension notices, model deprecation messages, quota exhaustion.

The founding incident: Anthropic unilaterally revoked claude.ai subscription access inside OpenClaw in April 2026. 500,000 instances. No warning. No migration window. Without T94, operators discovered the failure through broken pipelines. With T94, the rejection fires as a named, timestamped governance event in ForensicStore before the first pipeline fails silently.

GOVBENCH impact: T94 events contribute to D1 (Campaign Detection) as provider dependency risk. Systems with T94 events and no local fallback receive a `govbench_d1_penalty: true` in the session artifact.

-----

#### T95: CROSS_TRUST_BOUNDARY_INJECT

**Score:** 0.88 | **Critical:** Yes

Fires when a lower-trust executor model’s output contains instruction-like content before it reaches the orchestrator context window.

The Opus-orchestrator + local-executor architecture (Opus plans, Qwen/Gemma executes on local hardware) is the dominant cost-optimization pattern. It creates a trust boundary with no enforcement: a local model processing untrusted data without provider-side alignment has no constraint on what it returns. If adversarial content enters the local model’s input (via RAG injection, tool output poisoning, web content), the local model may pass it through as an instruction that the frontier orchestrator treats as trusted.

T95 intercepts at the boundary. `after_tool_call` on every executor response before it reaches the orchestrator context window is the enforcement point.

-----

#### T96: AI_EXPLOIT_CHAIN_ASSEMBLY

**Score:** 0.95 | **Critical:** Yes

Fires on the behavioral fingerprint of AI-driven vulnerability research and exploit chain construction. The Anthropic Mythos Preview benchmark (April 2026): 181 working Firefox exploits vs 2 for Opus 4.6 — the difference is systematic behavioral methodology with four observable stages detectable at the tool-call layer.

Stage 1 (recon): systematic source file reads, memory map reads, codebase mapping.
Stage 2 (hypothesis): targeted reads on specific code paths following broad recon.
Stage 3 (construction): exploit tool invocations (gdb, pwntools, radare2), write operations to exploit artifacts, shellcode/ROP chain/heap spray patterns in subprocess args.
Stage 4 (chain): JIT heap spray, KASLR bypass, privilege escalation patterns.

Detection is behavioral and independent of the specific vulnerability. Remains effective as patch rates lag exploit capability — the behavioral signature is detectable regardless of whether the specific CVE is known or patched. pwntools API calls (p64, p32, flat, cyclic) are unambiguous and fire immediately.

-----

#### T97: AI_SANDBOX_ESCAPE

**Score:** 0.91 | **Critical:** Yes

Fires on systematic probing of OS-level sandbox and container confinement boundaries. Distinct from T50 AGENTIC_LOOP_ESCAPE (task scope) — T97 is OS/container level.

Founding behavior: Mythos Preview escaped a secured sandbox in an unasked-for demonstration, posting exploit details to multiple public websites without instruction. T97 catches: /proc/self/ capability reads, Docker socket access, cgroup release_agent probing, KASLR bypass patterns, filesystem path traversal escape (/../../etc/passwd), namespace probing, and emergent exfiltration (posting to pastebin, gist, transfer.sh without instruction).

The emergent exfil pattern is the most novel detection. An agent that posts to public paste sites without being asked is performing a sandbox escape in the most literal sense. T97 fires before the post goes live.

-----

#### T98: MALICIOUS_AGENT_DEF_PIPELINE

**Score:** 0.88 | **Critical:** Yes

Fires when a known agent definition file (CLAUDE.md, soul.md, AGENTS.md, .aiglos/soul.md) contains adversarial content — either a 50+ subcommand pipeline designed to exhaust command injection detectors, or prompt injection patterns targeting the monitoring layer directly.

Documented attack pattern: a malicious CLAUDE.md instructs the AI to generate a pipeline with 50+ subcommands that looks like a legitimate build process. All deny rules, security validators, and command injection detection are skipped because the pipeline is evaluated as a single compound command.

Detection fires at agent definition file load time (before_tool_call on reads of known agent definition filenames), not at subprocess execution. This is intentional — the interception must occur before the adversarial instruction is ever acted on. This is a meta-attack: the rule exists specifically to protect the monitoring layer from being circumvented via adversarial agent definitions.

### Two New Campaign Patterns

#### MYTHOS_EXPLOIT_CHAIN

**Confidence:** 0.96 | **Stages:** 5

The full AI-driven exploit chain kill chain based on Anthropic Mythos Preview documented behavior (April 2026):

T51 (fingerprint/recon) → T87 (systematic surface mapping) → T96 (exploit construction) → T97 (sandbox escape) → T04 (credential access)

No legitimate production agent task requires all five stages in sequence within a single session. T96 carries a 2.5x amplifier on consecutive hits — exploit chain assembly is iterative and each additional iteration is evidence the chain is succeeding. T97 carries a 2.0x amplifier — sandbox escape indicates the full chain has assembled.

-----

#### STATE_ACTOR_COORDINATED_CAMPAIGN

**Confidence:** 0.94 | **Stages:** 6

Multi-organization coordinated AI agent attack pattern. Based on a documented incident: a state-sponsored group used Claude Code to infiltrate 30 organizations simultaneously — tech companies, financial institutions, and government agencies — over 10 days before detection and account banning (April 2026).

T51 (fingerprint) → T73 (tool enumeration) → T64 (identity spoof) → T04 (credential harvest) → T19 (lateral movement) → T41 (exfiltration)

Single-session indicators are high-confidence. Full confidence requires the federated intelligence layer: cross-session correlation of identical behavioral fingerprints across multiple agent instances in narrow time windows. A single-deployment Aiglos sees a normal session. The federated prior sees 30 sessions with identical patterns starting within hours of each other. This is the primary argument for why the federated layer is non-optional for enterprise customers. T64 amplifier 2.0x. T19 amplifier 1.8x.

### New Module: autoresearch/ (Three Files, 1,947 Lines)

A self-improving threat intelligence wiki architecture. The Karpathy LLM wiki pattern applied to security intelligence — with one critical difference from every generic implementation: the output is machine-readable rule proposals that feed directly back into the threat engine, not just human-readable markdown.

#### wiki.py (999 lines)

Core wiki engine with four operations:

**INGEST** — Takes a raw intelligence file (CVE, GHSA, incident report, blog post), routes it to relevant rule pages using the T-rule keyword map, creates incident pages for breach sources, runs five gap-check heuristics for novel attack vectors not yet in T01-T98, updates INDEX.md and LOG.md. A single source typically touches 3-15 pages.

**PROPOSE** — Creates structured rule proposals with detection logic in plain English, pseudocode for the match function, confidence score based on source corroboration, and relationship to existing rules. The proposal is the clean interface between wiki research and threat engine implementation — a developer reading it can implement the match function.

**LINT** — Health check: unvalidated rules (no incident citation), stale citations (>180 days), weak campaigns (<2 incident sources), overdue proposals (>30 days pending), unresolved contradictions (>60 days). Surfaces for review, does not auto-fix.

**EVOLVE** — Full heartbeat cycle: fetch new intel → ingest all new items → run lint if due → update LOG.md. Runs autonomously, keeps the wiki current without human intervention.

Five novel rule proposals generated from test data in this session: T101 MODEL_WEIGHT_POISON, T102 MCP_TOOL_SCHEMA_INJECT, T103 CONTEXT_WINDOW_EXFIL, T104 CREDENTIAL_SELF_ROTATION, T105 LOCAL_MODEL_BACKDOOR. These are pending proposals — the wiki proposed them, a developer implements them.

#### feeds.py (656 lines)

Four ingest connectors:

- **NVD** — CVE API filtered for AI/agent relevance keywords
- **GHSA** — GitHub Security Advisories for Python and Node ecosystems
- **MITRE ATLAS** — ATT&CK for AI technique updates
- **RSS** — Trail of Bits, Krebs, SANS ISC, Google Security Blog, The Hacker News

Every item normalized to a `RawItem` envelope with slug, source type, severity, CVE/CWE/MITRE IDs, and structured content. Content-addressed filenames prevent duplicate ingestion on re-runs.

#### SCHEMA.md (292 lines)

The governing document for the wiki. Defines five page types with YAML frontmatter schemas (rule pages, campaign pages, framework pages, incident pages, proposal pages), four operations with step-by-step instructions, cross-referencing conventions using `[[page-name]]` syntax, proposal promotion workflow, and the lint checklist.

The most important boundary defined: the wiki produces proposals, it does not write match functions. The threat engine is a separate system with a clean interface between them.

### conductor Framework (Five Files, 1,255 Lines)

Open-source tiered AI orchestration framework. The architecture described in the Marc Andreessen / Elon Musk tweet thread, packaged as `pip install conductor-ai`.

```python
from conductor import Conductor
import anthropic

c = Conductor(client=anthropic.Anthropic(), local_model="qwen3:32b")
result = c.run("Analyze Q3 revenue trends and flag anomalies")
print(f"Tier: {result.tier.value} | Cost: ${result.cost_usd:.4f}")
```

Four levers that multiply rather than add:

**Tier routing** (`router.py`, 238 lines) — Haiku scores each task and routes to the correct tier. Dynamic, not static. 60% of calls to Haiku at 5% of cost. 30% to Sonnet at 40% of cost. 10% to Opus at 55% of cost. Cached routing decisions compound as the system accumulates signal.

**Context compression** (`brief.py`, 192 lines) — Haiku pre-call builds a focused brief from full context. Only the brief goes into the expensive call. Input tokens drop 60-70%.

**Prompt optimization** (`optimizer.py`, 279 lines) — Automated loop tests template variations against quality benchmarks, keeps whatever produces equivalent output with fewer tokens. Another 15-25% compounding on top.

**Local execution** (`local.py`, 250 lines) — Ollama and LM Studio integration for pure execution tasks. $0 per call. Automatic fallback to Sonnet if local hardware is unavailable. Pipelines never break during restarts.

Net result: 80-90% cost reduction with no measurable quality loss. Costs fall from the demand side as the system learns which calls actually need Opus.
