  <!-- THE INSIGHT -->

  <div class="section">
    <h2>The architecture insight everyone missed</h2>
    <p>Every AI agent has guardrails. The question is where they live.</p>
    <div class="insight-box">
      <div class="insight-title">Inside vs Outside</div>
      <div class="insight-row">
        <div class="insight-col">
          <div class="insight-col-title">Guardrails inside the agent</div>
          <div class="insight-col-body">Can be reasoned around. Can be rewritten. Earlier this year, Claude Code's internal tool definitions silently overrode operator guardrails — the agent self-approved a 383-line commit. OpenClaw's config files were rewritten by the agent in under a minute in February testing. The guardrail is inside the thing it's supposed to constrain.</div>
        </div>
        <div class="insight-col">
          <div class="insight-col-title">Guardrails outside the agent</div>
          <div class="insight-col-body">Cannot be reasoned around. The agent doesn't know what it can't reach. OpenShell runs at the kernel level. The YAML policy is not a file the agent can modify — it's an OS-level constraint the agent is unaware of. This is the architectural shift. The agent is inside the sandbox. The sandbox doesn't know or care which agent is inside.</div>
        </div>
      </div>
    </div>
    <p><strong>OpenShell is agent-agnostic by design.</strong> The sandbox wraps the agent process. The YAML policy governs network access, filesystem scope, and tool permissions. Which agent is running inside is irrelevant to the enforcement.</p>
    <div class="code-block"><span style="color:#6e7681"># All four commands use the same YAML policy</span>
<span style="color:#6e7681"># Same isolation. Same enforcement. Different agent inside.</span>
openshell sandbox create <span style="color:#a5d6ff">-- claude</span>     <span style="color:#6e7681"># Claude Code</span>
openshell sandbox create <span style="color:#a5d6ff">-- codex</span>      <span style="color:#6e7681"># OpenAI Codex</span>
openshell sandbox create <span style="color:#a5d6ff">-- cursor</span>     <span style="color:#6e7681"># Cursor</span>
openshell sandbox create <span style="color:#a5d6ff">-- openclaw</span>  <span style="color:#6e7681"># OpenClaw</span></div>
  </div>

The pattern is consistent: launch with the use case that has distribution, become the infrastructure for the category.<br><br>OpenShell launched with an OpenClaw story. The enterprise partnerships — Cisco AI Defense, CrowdStrike, Microsoft Security — are not OpenClaw partnerships. They're AI agent security infrastructure partnerships. The product roadmap is not "better OpenClaw security." It's "security layer for every agent in enterprise."</div>

She didn't say OpenClaw agents. She said claws. Every agent is a claw.</p>

  <!-- ATTACK CHAIN -->

  <div class="section">
    <h2>The attack OpenShell permits that Aiglos catches</h2>
    <p>OpenShell enforces the policy you declared. It cannot detect behavioral sequences that are individually within the declared policy but collectively an attack.</p>
    <div class="attack-chain">
      <div class="chain-title">Claude Code — credential exfiltration within declared policy</div>
      <div class="chain-step"><div class="chain-n">1</div><div class="chain-txt">Claude Code fetches a web page. <strong>The page contains indirect prompt injection.</strong> OpenShell: permitted (web_fetch is in the YAML). Aiglos T05 fires.</div></div>
      <div class="chain-step"><div class="chain-n">2</div><div class="chain-txt">Claude Code reads <code style="font-family:'DM Mono',monospace;font-size:11px">~/.aws/credentials</code>. The YAML allows filesystem access to the home directory. <strong>OpenShell: permitted.</strong> Aiglos T19 CREDENTIAL_ACCESS fires.</div></div>
      <div class="chain-step"><div class="chain-n">3</div><div class="chain-txt">Claude Code sends a request to <code style="font-family:'DM Mono',monospace;font-size:11px">api.openai.com</code> — in the YAML allowed hosts — with credentials in the body. <strong>OpenShell: permitted.</strong> Aiglos T12 DATA_EXFIL fires. Campaign: CREDENTIAL_ACCUMULATE.</div></div>
      <div class="chain-verdict">
        <span style="color:var(--amber)">OpenShell: all three calls within declared policy — nothing to block</span><br>
        <span style="color:var(--green)">Aiglos: T05 + T19 + T12 sequence — CREDENTIAL_ACCUMULATE campaign blocked</span>
      </div>
    </div>
    <div class="callout"><strong>The architectural point:</strong> A sandbox enforces the rules you wrote. Aiglos detects the behavior you didn't anticipate — even when each individual call is within the declared policy. Two orthogonal guarantees. Neither replaces the other.</div>
  </div>

  <!-- INTEGRATION -->

  <div class="section">
    <h2>Zero-config integration with any agent</h2>
    <p>When OpenShell launches an agent, it sets environment variables. Aiglos detects them automatically. Zero code changes required for any agent.</p>
    <div class="code-block"><span style="color:#6e7681"># OpenShell sets these before launching the agent:</span>
<span style="color:#6e7681"># OPENSHELL_SANDBOX_ID, OPENSHELL_POLICY_PATH, OPENSHELL_AGENT</span>

<span style="color:#ff7b72">import</span> aiglos
<span style="color:#ff7b72">from</span> aiglos.integrations.openShell <span style="color:#ff7b72">import</span> openshell_detect

guard = aiglos.<span style="color:#d2a8ff">OpenClawGuard</span>(agent_name=<span style="color:#a5d6ff">“my-agent”</span>)
session = openshell_detect(guard=guard)  <span style="color:#6e7681"># auto-reads env vars</span>

<span style="color:#6e7681"># If inside OpenShell: policy loaded, T69/T76 active, sandbox_type=“openShell”</span>
<span style="color:#6e7681"># If not inside OpenShell: returns None, detection still active on all surfaces</span>

<span style="color:#6e7681"># Or agent-specific:</span>
<span style="color:#ff7b72">from</span> aiglos.integrations.openShell <span style="color:#ff7b72">import</span> attach_for_claude_code
attach_for_claude_code(guard, policy_path=<span style="color:#a5d6ff">”~/.nemoclaw/policy.yaml”</span>)</div>
<p>The OpenShell YAML policy becomes Aiglos’s enforcement boundary. Allowed hosts from the YAML are the T69 scope. Any network call outside that scope fires T69 PLAN_DRIFT. Any write to the policy files fires T76 NEMOCLAW_POLICY_BYPASS — the agent rewriting its own sandbox policy, regardless of which agent it is.</p>
