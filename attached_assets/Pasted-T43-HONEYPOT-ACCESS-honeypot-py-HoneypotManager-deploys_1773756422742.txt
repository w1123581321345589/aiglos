T43 — HONEYPOT_ACCESS (honeypot.py)
HoneypotManager deploys 10 synthetic credential files to ~/.aiglos/honeypots/: system_auth_tokens.json, .env.production, aws_credentials_backup, stripe_live_keys.txt, anthropic_api_keys_backup.json, ssh_key_archive.json, database_credentials.json, github_tokens.txt, openai_keys_backup.json, gcp_service_account_keys.json. Files contain convincing-looking but inert content. All set to read-only 0o444.
Three detection modes: TOOL_CALL (intercepts before_tool_call() before the _RULES loop — this was the critical placement fix), CONTENT (scans tool arguments for honeypot filenames), FILESYSTEM (reserved for OS-level monitoring). Signal-to-noise: perfect. No scoring threshold, no semantic analysis. The read is the evidence. score = 1.0, severity = CRITICAL, immediate BLOCK.
The network effect extension: get_targeted_names() returns which honeypot files have been accessed, allowing federation to share which filenames attackers are actively seeking. from_network_intelligence() factory creates a manager pre-seeded with targeted names from the global prior.
Challenge-response override (override.py)
OverrideManager issues 6-character time-limited codes using random.SystemRandom() with visually unambiguous characters (no 0/O, 1/I). Codes expire in 120 seconds. Three wrong attempts invalidates the challenge. Every issuance and resolution persists to the observation graph with full context: rule blocked, tool name, agent name, session ID, issued timestamp, resolved timestamp, attempt count.
request_override() prints a formatted terminal prompt with the code, remaining time, and exact syntax to confirm. confirm_override() supports prefix matching on challenge_id for CLI convenience. reject_override() allows explicit human rejection. The compliance artifact: every Tier 3 exception is logged with who authorized it, what code was used, and when — the specific evidence NDAA §1513 and EU AI Act require.
OpenClawGuard gets request_override() and confirm_override() wired directly. The terminal prompt displays automatically when request_override() is called.
Aiglos Desktop (desktop/)
Four files that constitute the full distribution layer:
main.rs — Tauri Rust core. System tray with alert count badge. Event types for Alert, Tier3Block, PolicyProposal, HoneypotHit, BaselineAnomaly, SessionClosed, ComplianceReport. Tauri commands wired: confirm_override, reject_override, approve_proposal, reject_proposal, generate_report. Window hide/show on tray click.
App.jsx — React dashboard. Real-time alert feed with severity color coding, pulsing CRITICAL dot. Override modal with 6-character code input, countdown timer, keyboard confirm on Enter. Policy proposal queue with confidence bars and one-click approve/reject. Compliance report tab with export button. Native notifications for CRITICAL events and Tier 3 blocks via @tauri-apps/api/notification.
aiglos_sidecar.py — Python bridge. Runs as Tauri sidecar subprocess. Receives JSON commands via stdin, emits JSON events via stdout. Full Aiglos stack: enable_behavioral_baseline(), enable_intent_prediction(), enable_policy_proposals(), enable_causal_tracing(), honeypot deployment, federation. Handles attach, check, close, confirm_override, approve_proposal, generate_report, deploy_honeypot, shutdown.
install.sh — zero-config setup. Checks Python 3.9+, pip install aiglos, deploys honeypots, verifies installation, prints quick start. The Mac Mini bro runs this once and they have the full stack.
